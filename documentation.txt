# Authentication & Banking API Documentation

Base URL: http://localhost/api

## 1. Authentication Endpoints

### Register
- **Path**: `/register`
- **Method**: `POST`
- **Summary**: Inscription d'un utilisateur
- **Request Body (JSON)**:
  - `name`: string (required)
  - `email`: string (required, unique)
  - `password`: string (required, min 8 chars)
- **Responses**:
  - `201 Created`: Account created successfully. Returns `message` and `token`.
  - `422 Unprocessable Entity`: Validation error.

### Login
- **Path**: `/login`
- **Method**: `POST`
- **Summary**: Connexion utilisateur
- **Request Body (JSON)**:
  - `email`: string (required)
  - `password`: string (required)
- **Responses**:
  - `200 OK`: Login successful. Returns `message` and `token`.
  - `401 Unauthorized`: Invalid credentials.

### Logout
- **Path**: `/logout`
- **Method**: `POST`
- **Summary**: Déconnexion utilisateur
- **Security**: Bearer Token (Sanctum)
- **Responses**:
  - `200 OK`: Logout successful.
  - `401 Unauthorized`: Unauthenticated.

## 2. Profile Endpoints (Auth Required)
*All these endpoints require a `Authorization: Bearer <token>` header.*

### Get Profile
- **Path**: `/me`
- **Method**: `GET`
- **Summary**: Récupérer les informations du profil
- **Responses**:
  - `200 OK`: Profile fetched successfully. Returns user data.

### Update Profile
- **Path**: `/me`
- **Method**: `PUT`
- **Summary**: Mettre à jour le nom ou l'email
- **Request Body (JSON)**:
  - `name`: string (sometimes required)
  - `email`: string (sometimes required, unique)
- **Responses**:
  - `200 OK`: Profile updated successfully. Returns updated user data.

### Update Password
- **Path**: `/me/password`
- **Method**: `PUT`
- **Summary**: Changer le mot de passe
- **Request Body (JSON)**:
  - `current_password`: string (required)
  - `new_password`: string (required, min 8 chars)
  - `new_password_confirmation`: string (required)
- **Responses**:
  - `200 OK`: Password updated successfully.
  - `422 Unprocessable Entity`: Current password incorrect or validation error.

### Delete Account
- **Path**: `/me`
- **Method**: `DELETE`
- **Summary**: Supprimer le compte utilisateur
- **Responses**:
  - `200 OK`: Account deleted successfully.

## 3. Account Endpoints (Auth Required)

### List Accounts
- **Path**: `/accounts`
- **Method**: `GET`
- **Summary**: Liste tous les comptes de l'utilisateur
- **Responses**:
  - `200 OK`: Accounts fetched successfully.

### Show Account
- **Path**: `/accounts/{id}`
- **Method**: `GET`
- **Summary**: Détails d'un compte spécifique
- **Responses**:
  - `200 OK`: Account details fetched successfully.

### Create Account
- **Path**: `/accounts`
- **Method**: `POST`
- **Summary**: Ouvrir un nouveau compte
- **Request Body (JSON)**:
  - `type`: string (checking | savings | minor)
  - `initial_balance`: numeric (optional)
  - `overdraft_limit`: numeric (optional, for checking)
  - `withdraw_limit`: integer (optional, for minor)
  - `guardian_id`: integer (optional, for minor)
- **Responses**:
  - `201 Created`: Account created successfully.

---

## 4. Transaction Endpoints (Auth Required)

### Deposit
- **Path**: `/accounts/{id}/deposit`
- **Method**: `POST`
- **Summary**: Effectuer un dépôt
- **Request Body (JSON)**:
  - `amount`: numeric (positive)
- **Responses**:
  - `200 OK`: Deposit successful.

### Withdraw
- **Path**: `/accounts/{id}/withdraw`
- **Method**: `POST`
- **Summary**: Effectuer un retrait
- **Request Body (JSON)**:
  - `amount`: numeric (positive)
- **Responses**:
  - `200 OK`: Withdrawal successful.
  - `422 Unprocessable Entity`: Insufficient funds or limit exceeded.

---

## 5. Transfer Endpoints (Auth Required)

### Internal Transfer
- **Path**: `/transfers`
- **Method**: `POST`
- **Summary**: Transférer de l'argent entre deux comptes
- **Request Body (JSON)**:
  - `source_account_id`: UUID
  - `destination_account_id`: UUID
  - `amount`: numeric (positive)
- **Responses**:
  - `200 OK`: Transfer completed successfully.
  - `422 Unprocessable Entity`: Logic error (funds, etc.)
  - `403 Forbidden`: Unauthorized (source account not owned by user)
